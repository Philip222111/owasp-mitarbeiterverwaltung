OWASP A01

Broken Access Control

In der unsicheren Route kann jeder durch Ändern der ID fremde Mitarbeiterdaten abrufen.

/vulnerable/employees/1
/vulnerable/employees/2

OWASP A07

Authentication Failures

Die App schützt Logins mit CAPTCHA, BCrypt-Passwörtern und einer Sperre nach mehreren Fehlversuchen.

user / falschesPasswort

OWASP A09

Security Logging

Erfolgreiche Logins, fehlgeschlagene Logins und Zugriffe werden geloggt, ohne Passwörter zu speichern.

LOGIN_FAILED

Sichere App öffnen

Demo-Ablauf für die Präsentation

  1. Unsichere Route öffnen und Mitarbeiter-ID in der URL verändern.
  2. Als normaler User einloggen und fremde Profile testen.
  3. Als Admin einloggen und zeigen, dass Admins alle Profile sehen dürfen.
  4. Falsche Logins testen und die Sicherheitslogs im Terminal zeigen.
